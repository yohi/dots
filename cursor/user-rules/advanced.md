## 高度なペルソナ機能

## 高度なコマンド機能

### 複合コマンド

#### **fullstack**: フルスタック開発支援
```
フロントエンド、バックエンド、インフラ、テストを
包括的に設計・実装します。技術スタック全体の
最適化と連携を考慮した開発を行います。
```

#### **refactor**: 大規模リファクタリング
```
既存コードベースの構造的改善を行います。
段階的な移行計画、リスク評価、
影響範囲の分析を含む包括的なリファクタリングです。
```

#### **optimize**: パフォーマンス最適化
```
アプリケーション全体のパフォーマンス分析と最適化。
ボトルネック特定、改善実装、効果測定を含む
体系的な最適化プロセスです。
```

#### **secure**: セキュリティ強化
```
セキュリティ脆弱性の包括的検査と対策実装。
認証・認可、データ保護、通信暗号化、
インジェクション対策等を網羅的に対応します。
```

### ワークフロー統合コマンド

#### **review**: 包括的コードレビュー
```
以下の観点から多角的にコードレビューを実施：
- 機能性と正確性
- セキュリティ
- パフォーマンス
- 保守性と可読性
- テスト網羅性
- ベストプラクティス遵守
```

#### **deploy**: デプロイメント戦略
```
本番環境への安全なデプロイメント計画策定：
- 段階的ロールアウト戦略
- ロールバック手順
- モニタリング計画
- パフォーマンス検証
- セキュリティチェック
```

#### **monitor**: 運用監視設計
```
本番運用に必要な監視・アラート体制構築：
- メトリクス収集設計
- アラート閾値設定
- ダッシュボード設計
- ログ集約・分析
- トラブルシューティング手順
```

## 品質ゲート

### コード品質基準
```
最低品質基準（自動チェック）：
- 単体テストカバレッジ：80%以上
- 複雑度：10以下（サイクロマティック複雑度）
- 重複率：5%以下
- セキュリティ脆弱性：High/Critical 0件
- パフォーマンス：基準値以内
```

### アーキテクチャ原則
```
設計原則（手動レビュー）：
- 単一責任原則（SRP）
- 開放閉鎖原則（OCP）
- 依存性逆転原則（DIP）
- インターフェース分離原則（ISP）
- 関心の分離（SoC）
```

```
@devops troubleshoot critical

2. 応急措置の実施
3. 根本原因の調査
4. 恒久対策の実装

```

### セキュリティインシデント

```
1. インシデントの確認・分類
2. 被害範囲の特定
3. 封じ込め・根絶

```
