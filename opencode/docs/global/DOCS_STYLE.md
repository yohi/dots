# Global Documentation Standards

ドキュメント（README, 仕様書, 設計書）作成時の共通ルールです。

## 1. 言語設定
- **本文**: 日本語 (Japanese)
- **技術用語**: 一般的な英単語はそのまま使用可（例: Request, Response, API, UI）。無理にカタカナにしない。

## 2. ファイル構成
- プロジェクトルートには、プロジェクトの目的を簡潔に記した `README.md` を必ず配置する。
- 詳細ドキュメントは `docs/` ディレクトリ配下に配置する。

## 3. 記述スタイル
- **結論ファースト**: 冒頭にそのドキュメントの目的と結論を書く。
- **Markdown活用**: リスト、引用、コードブロックを適切に使用し、可読性を高める。
- **コードブロック**: 必ず言語指定を行う（例: \`\`\`typescript）。

## 4. エージェントへの指示
ドキュメントを生成・更新する際は、既存のファイル構造と命名規則を尊重すること。
